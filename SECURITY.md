# Security & Privacy Policy

## Grundprinzipien
- Privacy by default
- Data minimization
- Secure-by-design

## Daten, die wir vermeiden
- Keine unnötige Speicherung personenbezogener Daten
- Keine API-Keys im Sourcecode oder in exportierten Dateien
- Keine versteckten Analytics-Tracker

## Sichere Entwicklung
- Regelmäßige Dependency-Audits
- Strikte Input-Validierung für Songdaten
- Defensive Behandlung externer APIs (Timeout, Retry, Rate-Limit)

## Responsible Disclosure
Bitte melde Security-Probleme verantwortungsvoll (kein public issue mit Exploit-Details).
